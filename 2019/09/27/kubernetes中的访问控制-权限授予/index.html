kubernetes中的访问控制-权限授予

Posted on 2019-09-27 | Post modified: 2019-10-03

kubernetes中访问控制的第一道关卡是用户认证，接下来来到第二道关卡：权限授予。

API请求的内容

在讨论权限授予之前，我们先看看发送至apiserver的请求中，包含了哪些内容。

示例：获取test命名空间的名为test的pod的信息

如上，该请求包括了：

  • token：275gX5MPWpNWYsHHoUePEFIvxo4uWAEs，使用token标识着k8s中的一个用户
  • url：https://169.254.128.15:60002/api/v1/namespaces/test/pods/test，包括apiserver地址，namespace，pod名。
  • HTTP请求方法：GET

不难看出，该请求可以分为主体，动作和资源三部分，即主体：主用户，动作：获取，资源：test命名空间下名为test的pod。实际上，k8s对API请求也进行了类似于主体，动作，资源的划分，但更为细化，大体如下：

  • user：访问的主体。
  • apiverb：对资源执行的操作，包括get、list、create、update、patch、watch、proxy、redirect、delete和deletecollection等。
  • resource：请求的目标资源的ID或名称。
  • namespace：目标资源所属的名称空间，仅对拥有命名空间属性的资源有效。
  • apiGroup：要访问的API组。默认为core API group。

如果请求的并非是资源的话，那么划分将有些变化，大体如下：

  • user：访问的主体。
  • requestpath：请求的非资源型路径，如/api或/healthz。
  • httpverb：要执行的操作，如get、post、put和delete。

此外，如果资源下面还有子资源的话，可以使用subresource表示子资源。

而权限授予，就是对主体用户授予对指定资源执行指定动作的权限方面的能力。

权限授予简介

在k8s中，为了核验用户的操作许可，成功通过身份认证后的操作请求还需要转交给授权插件进行许可权限检查，以确保其拥有执行相应的操作的许可。

apiserver主要支持如下授权插件：

  • Node：一种特殊的授权模式，专门授权由kubelet访问的API请求。
  • ABAC：Attribute-Based Access Control，基于属性的访问控制。
  • RBAC：Role-Based Access Control，基于角色的访问控制。
  • Webhook：基于HTTP回调机制通过外部REST服务检查确认用户授权的访问控制。
  • AlwaysAllow：不进行权限限制，即授予所有权限。
  • AlwaysDeny：拒绝所有权限的授予，仅用于测试。

目前主流的授权插件是RBAC，针对kubelet也可同时使用Node插件。

可以在API Server启动时，传入--authorization-mode=MODE参数来指定授权插件，可同时选择多个授权插件。

接下来说说k8s中主流的两种权限授予插件：ABAC和RBAC。

ABAC模式

ABAC（Attribute-Based Access Control，基于属性的访问控制）通过匹配策略对象来实现权限授予，每一个策略对象都包含多种属性：

  • version属性：表示“我是策略对象”，是策略对象的元数据，包括固定的apiVersion和kind。
  • subject-matching属性：表示发送访问请求的用户或组，包括user和group，两者选其一。
  • resource-matching属性：表示要访问的资源对象，包括apiGroup，namespace，resource。
  • non-resource-matching属性：表示要访问的非资源对象，包括nonResourcePath，即该非资源对象的路径。
  • readonly属性：表示是否只读，只读操作包括get，list和watch。

RBAC模式

RBAC（Role-Based Access Control，基于角色的访问控制）可以说是ABAC模式的改进版，比起ABAC模式直接将规则分配给用户，RBAC模式则是将规则分配给角色，然后再将角色应用至用户。

在ABAC模式中，新增策略需要通过写入策略文件然后重启apiserver，而RBAC模式则能够在运行中添加策略。为此，k8s新增加了两对资源对象：role和rolebinding，clusterrole和clusterrolebinding。

小结

  • 在k8s中，权限规则往往包括主体，对象，动作，用以描述api请求进行的操作。
  • k8s内置了多种权限插件，包括ABAC，RBAC，Webhook，Node以及往往用于测试的AlwaysAllow和AlwaysDeny，目前主流的插件是RBAC。
  • RBAC模式使用了role，clusterrole，rolebinding，clusterrolebinding四种资源对象，用以动态变更权限策略。
